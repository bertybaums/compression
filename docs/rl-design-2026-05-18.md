# Reasoner RL design (post-SFT-v2)

## Status

This plan is **gated on SFT v2 passing the milestone-eval criteria**:
1. Parse-success rate on v2 outputs ≥ 95% across stress / cx-patched / holdout benches.
2. Stress engagement ≥ 1.0 (up from v1's 0.13).

If either fails, fall back to special-token bracketing (see
`docs/sft-v2-design-2026-05-18.md`) before RL, OR abandon think-answer
structure entirely.

## What the May-16 plan said (and what changed)

The May-16 plan (see `memory/target_probe_rl_decision.md`) locked:

- **Reward**: `0.6 × substance + 0.2 × engagement + 0.1 × coherence + 0.1 × expressive_adequacy`
- **Reference target**: P4 (English-reason → translate to UGF)
- **Reward signal validation**: P4 wins P5 87.5% pairwise on a 50-prompt probe → the weighted reward tracks human preferences.

What changes with SFT v2 in place:

| Aspect | May-16 plan | v2-adapted (this doc) |
|---|---|---|
| Reward target | Full ugf_response | **Answer span only** (extracted via `eval.parse_v2_answer.extract_answer`) |
| Structural gate | UGF compliance | UGF compliance AND `parse_ok=True` (single marker present, non-empty answer) |
| Reference strategy | P4 as target | **Option A: self-play with CoT corpus SFT-mix anchor** (this doc) — see "Option B fallback" below |
| Algorithm | GRPO or REINFORCE+SFT-mix | **GRPO** — natural fit for sample-many-rollouts on the think-answer split |
| Reward scale | Per-rollout judge call | Same, but on answer span. `{0, +1}`-style not `{-1, +1}` per Numerals lesson |

The locked **reward weighting** (0.6/0.2/0.1/0.1) does NOT change. The
target-probe validation still applies — that was a head-to-head on
short-form generations, which our answer span is.

## Locked design (v2-adapted)

### 1. Reward function

```
reward(rollout) =
    0           if not is_valid_structure(rollout)
    0           if has_doom_loop(rollout)
    score       otherwise

where
    is_valid_structure(rollout) =
        validate_ugf(rollout).is_compliant
        AND parse_v2_answer.extract_answer(rollout).parse_ok

    has_doom_loop(rollout) =
        any n-gram of size N=4 appears >= R=3 times consecutively
        (deterministic rule applied to the full rollout text)

    score = 0.6 × substance(answer) +
            0.2 × engagement(answer) +
            0.1 × coherence(answer) +
            0.1 × expressive_adequacy(answer)
```

Where each dimension is the judge's 0–4 score normalized to [0, 1] (i.e.,
divide by 4), and score is therefore in [0, 1]. Reward is in [0, 1] for
structurally-valid rollouts, 0 for invalid ones.

**Why the doom-loop gate**: SFT v2 step-20000 sample probe (May 19) caught
a clear instance of the Labonne doom-looping pattern (`tlm-playbook/5_preference.md`):
the model repeated a philosophical conditional verbatim 8+ times instead
of developing it. A judge could score that "coherent" since each individual
sentence is grammatical — so the dimension-wise rubric scores can miss it.
A deterministic n-gram rule catches the surface form cheaply.

The playbook's prescribed two-stage mitigation is on-policy DPO + jury
filtering, followed by RL with n-gram repetition penalty. We're skipping
the DPO stage (GRPO + judge gives a directly-on-policy signal already)
but adopting the n-gram penalty for RL.

**Why score-on-answer (not full rollout)**: the think-answer separation in
v2 makes the reasoning the model's private scratch space. Judging the full
rollout would reward the reasoning's surface qualities (length, style)
independently of whether the model actually answered the prompt. Judging
the answer span tests the actual claim the model commits to — which is
what the project's expressive-adequacy thesis is about.

**Why `{0, +1}` not `{-1, +1}`**: Numerals' format-collapse incident at
small scale. With negative rewards on structural failures, the model can
get trapped in a regime where every rollout gets the same negative reward
and the gradient is zero. With `{0, +1}`, increasing the rate of valid
rollouts is always positively rewarded.

### 2. Algorithm: GRPO

Group Relative Policy Optimization, per DeepSeek-R1 and Ramji et al. 2026
"Thinking Without Words" stage 2.

Per prompt:
1. Sample N rollouts from the current policy (with temperature ~0.8).
2. Compute reward for each (see above).
3. Compute group-relative advantage:
   `A_i = (r_i - mean(r_group)) / (std(r_group) + ε)`
4. Policy gradient update with PPO-style clipping on the advantages.

Plus **SFT-mix anchor** (per Numerals lesson — see writeup.md):
```
total_loss = grpo_loss + λ_sft × CE(sft_example)
```
The SFT example is drawn from the same v2-formatted CoT corpus the SFT v2
training used. The anchor prevents catastrophic forgetting of the
think-answer format and the broader UGF reasoning style.

KL penalty against the reference policy (v2-SFT final) provides an
additional regularizer. Approximate KL via the standard GRPO formulation
(no separate value model needed).

### 3. Reference strategy: Option A (self-play with anchor)

Rollouts come from the v2 model itself. No external reference distribution.

The **CoT corpus serves as anchor data** via the SFT-mix term. The audit
(May 18, see `memory/corpus_audit_2026-05-18.md`) showed CoT-corpus quality
is ~3.86/3.80/3.27/3.67 on the rubric — comfortably above v1 Reasoner —
so it's a defensible anchor.

The P4 target-probe finding (87.5% over P5) serves as **reward-signal
validation**: the weighted reward correlated with human-preferred
rollouts on the probe. We're using that confidence in the reward function,
not P4 outputs as training targets.

### 4. Structural gate (compliance + parse_ok)

Every rollout passes through:
1. `validate_ugf(rollout)` → must be `compliant=True`.
2. `extract_answer(rollout)` → must have `parse_ok=True`.

Failures get reward = 0. We track per-step:
- `parse_failure_rate`: fraction with `parse_ok=False`
- `vocab_failure_rate`: fraction with UGF compliance failure
- `mean_reward`: average over the full batch

If `parse_failure_rate` is monotonically increasing across multiple steps,
RL is regressing on format. Halt and either (a) restart from earlier
checkpoint with stronger SFT-mix anchor, OR (b) fall back to special
tokens. We pre-register: **if parse_failure_rate exceeds 15% sustained
over 50 steps, halt the run.**

### 5. Judge: jury voting

Per the playbook's recommendation for subtle small-model failure modes
(`tlm-playbook/5_preference.md`): **3 parallel Claude subagent judges per
rollout, mean of their per-dimension scores**. This is cheap (parallel
Agent calls) and robust against single-judge idiosyncrasies — particularly
relevant for failure modes like doom-looping where a single judge might
score grammatically-valid loops as coherent.

Implementation: judge client dispatches 3 Agent calls in a single round,
collects 4-dim score JSON from each, computes mean per dim, returns the
weighted aggregate. If 1 of 3 judges fails to return parseable JSON, use
the mean of the remaining 2. If 2+ fail, retry the whole judging round
(this is rare with Claude subagents).

The jury also reduces variance in the reward signal, which is important
for GRPO advantage normalization — high judge variance inflates advantage
magnitudes and destabilizes training.

### 6. Judge bottleneck and rollout schedule

The 200 rpm MR cap is not the bottleneck here — Claude subagent judges
don't go through MR. The bottleneck is Claude subagent dispatch latency
(~3-4 min per batch of 30 items). At N=8 rollouts × 4 prompts/batch
× 3 jurors = 96 judge calls per step:

- 100 steps = 9,600 judge calls → can be parallelized across 12-24
  concurrent Agent dispatches; expect ~2-3h
- 1,000 steps = 96,000 → ~25-30h judge time end-to-end

Realistic schedule for a smoke-test RL run:
- 8 rollouts × 4 prompts/batch × 3 jurors = 96 judge calls per step
- 100 steps total
- ~2-3h judge time + ~1h GPU time = ~3-4h

Production RL run: scale to 1,000-2,000 steps after smoke-test validates
signal. May need to cut jury size to 2 or run shorter to keep judging
tractable.

### 7. Logging and diagnostics

Per-step tensorboard / log:
- `reward/mean`, `reward/std`, `reward/min`, `reward/max`
- `reward/structural_fail_rate` (parse + vocab combined)
- `reward/doom_loop_rate` (n-gram rule fired)
- `reward/per_dimension` (judge means for engage/coher/subst/expr)
- `reward/jury_disagreement` (per-dim stdev across the 3 jurors)
- `train/policy_loss`, `train/kl_penalty`, `train/sft_anchor_loss`
- `train/lr`, `train/grad_norm`
- Per ~50 steps: sample 3 rollouts to a samples log, with full text

## Option B fallback (documented for future reference)

If self-play RL fails to improve over SFT v2 — specifically if
mean-reward plateaus below the v2-SFT baseline answer-span scores, OR if
the model drifts toward an off-corpus distribution — revisit Option B:

> **Option B: Regenerated P4 references in v2 format**
> - For each RL training prompt, generate a "P4-v2" reference: English
>   reasoning via gpt-oss-120b at medium reasoning_effort, then split the
>   English reasoning into reasoning + final claim, translate each to UGF
>   via `translator/mr_structured_translator.py`, wrap with the marker.
> - Add this as a fixed reference distribution alongside the SFT-mix
>   anchor — either as a separate CE term (`λ_ref × CE(ref_example)`) or
>   as an "ideal completion" that the policy is regularized toward.

Reasons we're not starting with Option B:
1. **Pipeline cost**: regenerating P4 references in v2 format requires a
   reasoning/claim split (heuristic) on the gpt-oss output before
   translation — adds noise and computation.
2. **Format drift risk**: the P4 references would have been generated by
   a teacher that doesn't natively use the marker. Wrapping its output
   post-hoc introduces stylistic artifacts (we already saw this in the
   May-14 P4 EA score of 2.84).
3. **Reward signal already validated**: P4 won 87.5% over P5 on the
   target probe. That gives us confidence the reward function (judging
   the answer dimension-wise) tracks human preference. We don't need P4
   ROLLOUTS to train against; we need the REWARD to be correct.
4. **Cleaner experimental design**: self-play RL with SFT anchor is a
   single-knob experiment; adding fixed references introduces a second
   distribution that confounds attribution of any observed improvement.

**Decision point to revisit Option B**: end of smoke-test RL run, if
mean-reward fails to beat the SFT v2 baseline by ≥ 0.05 (on a 0-1 scale)
after 100 steps.

## Hyperparameters (initial, to be tuned)

- learning rate: 1e-6 (low, since we're refining a converged model)
- batch size: 4 prompts × 8 rollouts = 32 sequences per gradient step
- max_new_tokens: 400 (same as bench generation)
- temperature: 0.8 (matches bench generation)
- top_p: 0.9, top_k: 50
- λ_sft (anchor weight): 0.5 (start conservative; tune based on whether reasoning style drifts)
- λ_kl (KL penalty): 0.05 (typical GRPO range; tune if policy collapses or fails to move)
- clip ratio: 0.2 (PPO standard)
- max_grad_norm: 1.0

## Pre-implementation checklist

1. SFT v2 milestone passes (parse-rate ≥ 95%, stress engagement ≥ 1.0).
2. Locked reward function spec (this doc).
3. Reward client wrapping the judge — async, rate-limited via the
   existing `AsyncTokenBucket` (corpus/generation/rate_limiter.py).
4. Rollout buffer: prompts in, sampled rollouts out, rewards back.
5. GRPO trainer (new: `reasoner/train_rl.py`).
6. RL config (new: `configs/reasoner_rl_v1.yaml`).
7. SLURM script (new: `slurm/train_reasoner_rl.sbatch`).
8. Eval pipeline reuses milestone scripts (`run_sft_v2_bench.py`,
   `prepare_v2_milestone_judge_batches.py`, `aggregate_v2_milestone.py`).

## Out of scope (for v1 of RL; revisit later)

- **Curriculum / staged RL**. Start with all prompts mixed; consider
  difficulty-staged later if convergence is slow.
- **Process reward** (rewarding the reasoning span). The answer-span
  reward already creates implicit pressure on the reasoning, since
  parse_ok requires the marker. Process reward would be a v3 step.
- **Multi-turn / interactive RL**. Single-turn only.
- **Tool use**. Single-turn pure-UGF only.

## Paper-draft note

A paragraph for the methods section:

> "We applied Group Relative Policy Optimization (GRPO; Shao et al. 2024)
> using a single weighted reward computed by an external LLM judge
> (Claude subagents against a 4-dimensional, anchored 0–4 rubric on the
> answer span only). The reward function — locked a priori based on a
> 50-prompt blinded head-to-head probe — gives 60% weight to
> philosophical substance, 20% to prompt engagement, and 10% each to
> coherence and UGF expressive adequacy. Structural rollouts (UGF vocab
> compliant; exactly one marker phrase; non-empty answer span) received
> rewards in [0, 1]; non-compliant rollouts received 0, following
> Numerals project lessons on avoiding zero-gradient traps in small-model
> RL. We used the SFT v2 model as both the policy initialization and the
> KL reference, with a 50% SFT-mix anchor on the CoT corpus to prevent
> distributional drift."
